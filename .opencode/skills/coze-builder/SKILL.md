# Coze 智能体构建指南

## Coze 平台概述

Coze 是字节跳动推出的 AI Agent 开发平台，支持创建 Bots、工作流、知识库等。

**国际版**: https://coze.com
**国内版**: https://coze.cn

## 核心概念

| 概念 | 说明 |
|------|------|
| Bot | AI 助手/智能体 |
| Workflow | 工作流编排 |
| Plugin | 插件（工具集成）|
| Knowledge | 知识库 |
| Agent | 多 Agent 模式 |

## 创建智能体的步骤

### 1. 明确 Bot 定位

- Bot 的核心功能是什么？
- 目标用户是谁？
- 需要哪些工具和能力？

### 2. 选择创建方式

- **空白创建**: 从零配置
- **模板创建**: 基于已有模板修改

### 3. 配置基础信息

- Bot 名称和描述
- 头像和简介
- 开场白（可选）

### 4. 设置人设与逻辑

编写详细的 Prompt，包含：
- 角色定义
- 能力描述
- 输出格式要求
- 约束条件

### 5. 添加功能组件

| 组件 | 功能 |
|------|------|
| Plugin | 集成第三方 API |
| Workflow | 复杂流程编排 |
| Knowledge | RAG 知识库 |
| Memory | 长期记忆 |
| Variable | 变量管理 |

## 多 Agent 模式

多 Agent 模式用于处理复杂任务，通过将任务分解为多个子模块，每个 Agent 负责特定任务。

### 适用场景

- 复杂对话流程
- 多轮任务处理
- 需要不同专业能力的任务

### 配置步骤

1. **创建 Bot 时选择「多 Agent 模式」**

2. **设置人设与逻辑**
   - 定义整体目标
   - 说明 Agent 协作方式

3. **添加节点并配置**
   - 每个节点是一个 Agent
   - 配置模型、适用场景
   - 编写节点 Prompt 和技能

4. **设置跳转条件**
   - 根据用户目标语言
   - 根据任务类型
   - 根据输出结果

### 示例：翻译器

```
用户输入 → 意图识别节点
           ↓
    ┌──────┴──────┐
    ↓             ↓
英文翻译节点   中文翻译节点
    ↓             ↓
返回结果      返回结果
```

**跳转条件设置**：
- 用户说 "translate to English" → 跳转到英文翻译节点
- 用户说 "翻译成中文" → 跳转到中文翻译节点

## 工作流节点

### 1. 开始节点

定义输入参数：
- 用户问题
- 上下文变量
- 文件上传

### 2. 大模型节点

配置：
- 选择模型
- 设置 Prompt
- 配置变量

### 3. 代码节点

执行自定义逻辑：
- 数据处理
- 格式转换
- 条件判断

### 4. 条件分支节点

根据条件分流：
- 意图判断
- 内容分类
- 参数验证

### 5. 插件节点

调用外部工具：
- HTTP 请求
- 第三方 API
- 自定义插件

### 6. 结束节点

输出结果：
- 文本回复
- 结构化数据
- 跳转链接

## 常用插件

- **DALL·E**: 图像生成

- **Text to Speech**: 语音合成

### 自定义插件
- 通过 API 集成第三方服务
- 创建自己的插件

## 测试与调试

### 预览模式

1. 点击「预览」进入测试页面
2. 输入测试问题
3. 查看执行过程和结果

### 调试技巧

1. **逐步测试**: 逐个节点测试
2. **查看日志**: 每个节点都有执行日志
3. **变量检查**: 查看中间变量值
4. **错误定位**: 红色标记错误节点

## 发布与使用

### 发布 Bot

1. 点击「发布」
2. 选择发布平台
3. 确认发布

### 使用方式

1. **Coze 平台**: 通过链接分享
2. **API 调用**: 获取 API Key 进行集成
3. **嵌入网站**: 通过 Embed 代码嵌入
4. **第三方平台**: Discord、Telegram 等

## 常见问题

### Q: 多 Agent 模式如何调试？
A: 在预览模式下查看每个 Agent 的输入输出，逐步排查问题。

### Q: 如何提高 Bot 响应速度？
A: 优化 Workflow 结构，减少不必要的节点，使用缓存。

### Q: 如何处理复杂任务？
A: 使用多 Agent 模式或工作流，将任务拆分为多个子任务。

### Q: Bot 如何访问外部 API？
A: 创建自定义 Plugin，配置 API 认证和参数。

### Q: 如何让 Bot 有记忆？
A: 启用 Memory 功能，或使用变量保存上下文。
