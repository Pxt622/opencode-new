# Dify 智能体构建指南

## Dify 平台概述

Dify 是一个开源的 LLM 应用开发平台，支持创建智能体（Agent）、工作流（Workflow）、RAG 等。

**官网**: https://dify.ai/
**开源仓库**: https://github.com/difyhq/dify

## 搭建智能体的步骤

### 1. 明确目标

在开始前，明确：
- 智能体的用途是什么？（客服、问答、内容生成、数据分析等）
- 目标用户是谁？
- 需要什么样的输出格式？
- 如何评估智能体的效果？

### 2. 架构设计

根据复杂程度选择：
- **简单场景** → 单一 LLM + Prompt
- **中等场景** → LLM + 工具（Tools）
- **复杂场景** → 多 Agent 协作 + 工作流

### 3. 组件选择

Dify 提供的核心组件：

| 组件 | 功能 | 适用场景 |
|------|------|----------|
| LLM | 大语言模型调用 | 核心推理 |
| Prompt | 提示词模板 | 结构化输出 |
| HTTP | 调用外部 API | 第三方集成 |
| Python | 自定义逻辑 | 数据处理 |
| Knowledge | 知识库检索 | RAG |
| Workflow | 流程编排 | 复杂任务 |
| Branch | 条件分支 | 逻辑分流 |
| Loop | 循环处理 | 批量任务 |

### 4. Workflow 建模

**基础工作流模式**：

```
用户输入 → 意图识别 → [分支1: 问答] → 回复
                    → [分支2: 执行任务] → 工具调用 → 结果处理 → 回复
                    → [分支3: 转人工] → 结束
```

**进阶模式**：
- 并行处理：多个分支同时执行
- 迭代循环：重复执行直到满足条件
- 子工作流：复用已定义的工作流

### 5. Prompt 编排与控制

**Prompt 结构建议**：
```
## 角色
你是一个[角色描述]

## 背景
[上下文信息]

## 任务
[具体任务描述]

## 约束
- 约束1
- 约束2

## 输出格式
[期望的输出格式]
```

**控制技巧**：
- 使用示例（Few-shot）引导输出格式
- 使用条件判断控制分支
- 使用变量传递上下文

### 6. 中间结果验证

- 设置检查点验证中间输出
- 配置错误处理和重试逻辑
- 添加日志便于调试

### 7. 监控与优化

- 监控运行日志和性能指标
- 收集用户反馈持续优化
- 定期更新知识库

## 从空白搭建示例

### 步骤 1：创建智能体
1. 登录 Dify 控制台
2. 点击「创建应用」→「空白应用」
3. 选择「智能体」或「工作流」

### 步骤 2：配置 LLM
1. 在画布中添加「LLM」节点
2. 选择模型（如 GPT-4、Claude）
3. 配置温度、最大 token 等参数

### 步骤 3：编写 Prompt
1. 点击 LLM 节点
2. 在「提示词」中编写系统提示
3. 设置用户提示模板

### 步骤 4：添加工具（可选）
1. 点击「添加」→ 选择工具类型
2. 配置工具参数
3. 在 Prompt 中调用工具

### 步骤 5：测试与发布
1. 在「预览」中测试
2. 调试直到效果满意
3. 点击「发布」

## 从模板搭建

Dify 提供多种模板：
- **AI Agent**: 带推理能力的智能体
- **Chatflow**: 对话式工作流
- **Workflow**: 复杂流程编排
- **Text Generator**: 文本生成
- **Realtime**: 实时对话

选择模板后，根据需求修改即可。

## 常见问题

### Q: 如何让 Agent 自动调用工具？
A: 在 Prompt 中明确指定工具名称和调用时机，Agent 会根据任务自动判断。

### Q: 如何处理多轮对话？
A: 使用「会话历史」变量或在 Workflow 中添加状态管理。

### Q: 如何提高输出稳定性？
A: 降低 temperature 参数，增加输出格式约束，提供示例。
